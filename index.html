Overview

MetalÂ³ (pronounced Metal Kubed) is a project that provides components to enable the management of bare metal hosts for Kubernetes. It functions as a Kubernetes application, meaning it is managed and operated through Kubernetes interfaces. It provides a unified API for managing bare metal resources, allowing users to easily provision, configure, and manage their physical servers and other bare metal resources. Metal3 also provides a unified platform for managing virtual machines, allowing users to easily provision, configure, and manage their virtual machines.

Goals

  1. To provide an open source, extensible, and production-ready bare metal provisioning and lifecycle management platform for Kubernetes.
  2. To enable users to quickly and easily deploy and manage bare metal clusters in a secure and reliable manner.
  3. To provide a platform that is easy to use and maintain, and that is highly available and scalable.
  4. To provide a platform that is compatible with existing Kubernetes distributions and tools.
  5. To provide a platform that is extensible and customizable to meet the needs of different users.
  6. To provide a platform that is secure and compliant with industry standards.
  7. To provide a platform that is cost-effective and efficient.

Mission

The mission of the metal3.io project is to provide an open source, extensible, and easy-to-use platform for managing bare metal servers. The project aims to provide a comprehensive set of tools and APIs to enable users to quickly and easily provision, configure, and manage bare metal servers. The project also seeks to provide a platform for the development of new features and capabilities for managing bare metal servers.

Components

  1. Machine Controller-Bare metal actuator

    2. The Bare Metal Actuator is the first element of the system, which is an implementation of the Machine Actuator interface from the cluster-api project. This actuator responds to changes in Machine objects and serves as a customer of the BareMetalHost custom resources managed by the Bare Metal Operator.

  2. Bare metal operator

    3. It includes a Controller for a new Custom Resource, BareMetalHost. This custom resource represents an inventory of known (configured or automatically discovered) bare metal hosts. When a Machine is created the Bare Metal Actuator will claim one of these hosts to be provisioned as a new Kubernetes node. In response to BareMetalHost updates, the controller will perform bare metal host provisioning actions as necessary to reach the desired state. The creation of the BareMetalHost inventory can be done in two ways: Manually via creating BareMetalHost objects. Optionally, automatically created via a bare metal host discovery process. For more information about Operators, see the operator-sdk.

  3. Bare metal management pods

    4. The operator manages a set of tools for controlling the power on the host, monitoring the host status, and provisioning images to the host. These tools run inside the pod with the operator, and do not require any configuration by the user.Bare metal management pods are used to manage the underlying physical infrastructure of a Kubernetes cluster. These pods provide a way to manage the physical resources of a cluster, such as servers, storage, networking, and other hardware components. They also provide a way to monitor and manage the health of the cluster, as well as to deploy and manage applications on the cluster. The pods can be used to automate the provisioning and configuration of the physical infrastructure, as well as to monitor and manage the health of the cluster.
